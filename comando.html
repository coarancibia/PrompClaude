01

La idea principal

Claude Code trabaja sobre la carpeta donde fue abierto.

Por eso, antes de escribir claude, siempre hay que confirmar que estamos dentro de la carpeta correcta del proyecto.

02

Dos formas de usar Claude

Ambas terminan en el mismo lugar: escribir claude dentro de la carpeta correcta.

Terminal del Mac

Cuándo usarla

Cuando quieres abrir Claude sin depender de VS Code. Es la app "Terminal" del sistema, independiente de cualquier editor.

Terminal de VS Code

Cuándo usarla

Cuando ya tienes el proyecto abierto en VS Code y quieres trabajar viendo los archivos mientras Claude hace cambios.

05

No me pierdo si...

Las cinco situaciones más comunes, y qué hacer en cada una.

Claude no encuentra archivos

Qué significaLo más probable es que abriste Claude en la carpeta incorrecta.
Qué revisarLa ruta que muestra pwd antes de abrir Claude.
Qué comando usarpwd y luego cd a la carpeta correcta.

Claude se confundió

Qué significaEstá mezclando cosas o arrastrando contexto viejo de la conversación.
Qué revisarSi el chat quedó muy largo o se desvió del tema original.
Qué comando usar/clear para partir más limpio.

Cerré Claude y quiero retomar

Qué significaCerraste la terminal o el proyecto y necesitas volver.
Qué revisarSi quieres continuar lo último o elegir una conversación anterior.
Qué comando usarclaude --continue (o -c) para lo último, claude --resume (o -r) para elegir.

Claude no abre

Qué significa"command not found: claude" — no está instalado o la terminal no lo reconoce.
Qué revisarCierra y abre la terminal de nuevo antes de asumir que falta instalar algo.
Qué comando usarReintenta claude; si persiste, revisa la instalación de Claude Code.

No sé si estoy en la carpeta correcta

Qué significaNo confirmaste tu ubicación antes de abrir Claude.
Qué revisarLa ruta que devuelve pwd: debe terminar en la carpeta del proyecto, no en algo genérico como /Users/constanza.
Qué comando usarpwd y ls para confirmar antes de escribir claude.
