01

Cómo funciona el proceso

Antes de convertirse en una skill instalable, cada propuesta recorre las mismas cinco etapas. Esta página cubre solo la primera: la propuesta visual.

Propuesta → Revisión → Piloto → Validación → Publicación
01

Se documenta la idea: objetivo, problema que resuelve y flujo propuesto.

02

El equipo revisa el flujo, los inputs y los outputs antes de avanzar.

03

Se prueba con casos reales y se ajusta según los resultados.

04

Se confirma que resuelve el problema sin reemplazar el criterio humano.

05

Se empaqueta como skill instalable y se comparte con el equipo.

Editar componentes de Figma con DS Prime

Skill de Claude Code para revisar, migrar o crear componentes en Figma utilizando correctamente el Design System Prime.

Nombre técnico editar-componente-figma-ds Design System Versión 5 Validada

Esta skill no reemplaza la revisión de una Product Designer. Todo componente debe validarse visualmente antes de reemplazar o publicar un componente oficial.

01

Descripción general

Actúa como especialista en Design Systems dentro de Figma, usando el Design System Prime como única fuente de verdad para migrar componentes existentes o crear átomos, moléculas y organismos nuevos.

DS Token · Prime

  • Variables primitivas y semánticas de color
  • Variables de spacing, padding y gap
  • Radios, bordes y strokes
  • Tamaños y efectos
  • Iconografía oficial
  • Configuración Light

DS Components · Prime

  • Text Styles oficiales
  • Átomos, moléculas y organismos
  • Component sets, variantes y propiedades
  • Componentes anidados

Fuentes oficiales guardadas en la skill: DS Token · Prime y DS Components · Prime.

02

Qué permite hacer

Migrar y crear

  • Migrar componentes existentes
  • Crear átomos, moléculas y organismos
  • Revisar si un componente ya existe antes de crear uno nuevo
  • Reutilizar átomos y moléculas como instancias vinculadas

Aplicar el Design System

  • Variables e iconografía desde DS Token · Prime
  • Text Styles y componentes desde DS Components · Prime
  • Modo Light sobre fondo blanco, siempre
  • Auto Layout, variantes, propiedades y comportamiento responsive

Cuidar la calidad

  • Revisar accesibilidad
  • Presentar una propuesta antes de editar
  • Trabajar sobre una copia y mantener intacto el original

Nunca hace

  • Detach de instancias de átomos o moléculas.
  • Editar el componente original.
  • Publicar una librería automáticamente.

Siempre hace

  • Trabaja sobre una copia.
  • Presenta una propuesta y espera aprobación.
  • Vincula componentes como instancias.
03

Requisitos previos

Cada Product Designer que use esta skill debe tener:

Entorno

  • Mac
  • Claude Code instalado
  • Acceso a Terminal
  • Figma MCP conectado a Claude Code

Acceso en Figma

  • Permiso de lectura y edición en Figma
  • Acceso a DS Token · Prime
  • Acceso a DS Components · Prime
  • DS Components publicado y habilitado como librería en el archivo de destino

La skill no reemplaza la revisión de una Product Designer. Todo componente debe validarse visualmente antes de reemplazar o publicar un componente oficial.

04

Descarga de la skill

Versión actual: editar-componente-figma-ds v5.

Descargar skill v5

Descarga el archivo editar-componente-figma-ds-v5.zip. Guárdalo en ~/Downloads o en ~/Desktop/Claude/skill/ antes de instalar.

05

Instalación desde Terminal

Elige el bloque según dónde guardaste el ZIP descargado.

Si el ZIP está en ~/Downloads
Terminal
mkdir -p ~/.claude/skills

rm -rf ~/.claude/skills/editar-componente-figma-ds

TEMP_DIR=$(mktemp -d)

unzip -q ~/Downloads/editar-componente-figma-ds-v5.zip -d "$TEMP_DIR"

SKILL_DIR=$(dirname "$(find "$TEMP_DIR" -type f -name "SKILL.md" | head -n 1)")

cp -R "$SKILL_DIR" ~/.claude/skills/editar-componente-figma-ds

rm -rf "$TEMP_DIR"

if [ -f ~/.claude/skills/editar-componente-figma-ds/SKILL.md ]; then
  echo "✅ Skill instalada correctamente"
else
  echo "❌ No se pudo instalar la skill"
fi
Resultado esperado: ✅ Skill instalada correctamente

Aprobación y calidad
12

Sistema de aprobación antes de editar

Nunca edita el original ni publica una librería automáticamente. Trabaja únicamente sobre una copia.

APROBAR COPIA AJUSTAR PROPUESTA: [INDICAR QUÉ DEBE CORREGIR] CANCELAR
13

Revisión tipográfica obligatoria

No basta con copiar fuente, peso, tamaño y line-height. Cada capa debe quedar vinculada al Text Style real de DS Components · Prime.

Prompt de corrección tipográfica
Claude Code
La migración tipográfica no está aprobada.

No crees otra copia y no modifiques otras propiedades.
Corrige únicamente la tipografía de la copia actual.

Antes de corregir:

1. Enumera todas las capas de texto de todas las variantes.
2. Indica el Text Style actual.
3. Indica el Text Style exacto de destino.
4. Confirma que DS Components · Prime está habilitado.
5. No utilices nombres aproximados.
6. No copies solamente valores tipográficos.
7. Debe existir un vínculo nativo con el Text Style.

No modifiques todavía.
Presenta la tabla y espera mi aprobación.
Acción de aprobación: APROBAR CORRECCIÓN TIPOGRÁFICA
14

Auditoría final

Prompt de auditoría final
Claude Code
Realiza una auditoría final del componente.

Confirma capa por capa:

- Text Style vinculado.
- Variables de color vinculadas.
- Variables de spacing vinculadas.
- Procedencia de cada icono.
- Componentes anidados utilizados.
- Variantes y propiedades.
- Valores locales pendientes.
- Estado de Auto Layout.
- Estado responsive.
- Estado de accesibilidad.
- Modo Light sobre fondo blanco.

No declares el componente terminado si queda algún incumplimiento.
15

Errores frecuentes

Claude no encuentra la skill+

Revisa que ~/.claude/skills/editar-componente-figma-ds/SKILL.md exista y que el archivo esté directo dentro de esa carpeta, no dentro de una carpeta anidada.

Claude no puede conectarse a Figma+

Ejecuta /mcp dentro de Claude para revisar el estado. Si Figma no aparece conectado, reinstala el plugin con claude plugin install figma@claude-plugins-official.

La tipografía no tiene Text Style+

Copiar fuente, peso, tamaño y line-height no es suficiente. Usa el prompt de revisión tipográfica de la sección 13 para vincular el Text Style real.

Los iconos provienen de otra librería+

Toda la iconografía debe provenir de DS Token · Prime. Si un ícono no existe ahí, repórtalo antes de continuar en vez de traerlo de otra librería.

El componente quedó diseñado para fondo oscuro+

La skill trabaja siempre en modo Light sobre fondo blanco. Si aparece contenido pensado para modo oscuro, indícalo para que se corrija antes de aprobar.

Los espaciados siguen siendo manuales+

Todo padding y gap debe estar vinculado a variables de spacing de DS Token · Prime, no a valores fijos escritos a mano.

Claude modificó el componente original+

Esto no debería ocurrir: la skill trabaja siempre sobre una copia. Si sucede, repórtalo y no publiques ni reemplaces el componente oficial.

El archivo SKILL.md quedó dentro de una carpeta incorrecta+

Vuelve a ejecutar la instalación desde la sección 05. El script busca SKILL.md con find y copia esa carpeta directamente a ~/.claude/skills/editar-componente-figma-ds.

16

Buenas prácticas

Hacer

  • Revisar si el componente o átomo ya existe antes de crear uno nuevo.
  • Reutilizar átomos y moléculas como instancias vinculadas.
  • Aplicar variables, Text Styles e iconografía oficial de los dos DS.
  • Trabajar siempre en modo Light sobre fondo blanco.
  • Presentar la propuesta y esperar aprobación antes de editar Figma.
  • Trabajar sobre una copia, dejando el original intacto.

No hacer

  • No hacer detach de instancias.
  • No duplicar componentes existentes.
  • No copiar solo valores tipográficos sin vincular el Text Style.
  • No traer iconos de librerías distintas a DS Token · Prime.
  • No editar ni publicar el componente original automáticamente.
