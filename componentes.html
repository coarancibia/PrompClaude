01

Contexto general del proyecto

Antes de pedir cualquier componente, el proyecto en Claude debe tener cargados los archivos de fuente de verdad. Esto se hace una sola vez y no se repite en cada chat.

Cargar en el proyecto de Claude: Estilos base (colores, tipografía, espaciados, radius, overlays, estados, variables) · Componentes existentes (comportamiento, variantes, anatomía, responsive, documentación) · Branding (logo, marca, acentos de color, convivencia con banderas Cencosud).
Regla principal del proyecto
Instrucciones del proyecto
Claude debe usar estos archivos como fuente de verdad y no inventar reglas visuales que no estén ya definidas.
Se agrega junto con los archivos cargados (Estilos base, Componentes existentes, Branding).
02

Modo creación de componentes

Este bloque se agrega al final de las instrucciones del proyecto. Le dice a Claude cómo comportarse cada vez que se le pide un componente con un link de Figma.

Bloque para instrucciones del proyecto
Instrucciones del proyecto · pegar al final
Modo creación de componentes en Figma:

Cuando pida crear un componente y entregue un link de Figma, crear el componente directamente en el lugar indicado.

Usar el contexto de los estilos, componentes existentes y branding cargados como fuente de verdad.

Respetar siempre:
tokens, auto layout, variables, estados, accesibilidad AA, nomenclatura del sistema y responsive.

El componente debe quedar funcional dentro del layout donde se inserta:
- No usar ancho fijo salvo que así esté definido.
- Usar Hug contents, Fill container o constraints según corresponda.
- Revisar el contenedor padre antes de ajustar el componente.
- Evitar posiciones absolutas innecesarias.
- El componente no debe salirse del frame ni romper el layout.
- Si el espacio no alcanza, debe adaptarse, bajar de línea o ajustar su contenido según corresponda.
- Mantener padding, gap, radius y tipografía según los estilos definidos.

Dejarlo armado como componente principal, con variantes/properties editables si corresponde.

No hacer análisis previo ni explicación larga. Priorizar ejecutar en Figma.
Se pega una sola vez, al final de las instrucciones del proyecto. No se repite en los chats.

04

Pedir un componente simple

Para componentes sin mayor complejidad, basta con nombre + link. Este es el prompt que se usa día a día dentro del chat "Componentes".

Link: [LINK DE FIGMA]

05

Pedir un componente con especificaciones

Usar esta estructura cuando el componente tenga interacción, estados o comportamiento responsive que no son obvios a partir del nombre.

Plantilla con especificaciones
Chat "Componentes"
Crear componente:
[NOMBRE DEL COMPONENTE]

Link:
[LINK DE FIGMA]

Especificaciones:
- Uso: [para qué sirve]
- Contenido: [textos, íconos, labels, opciones]
- Comportamiento: [seleccionable, desplegable, editable, cerrado/abierto, etc.]
- Estados: [default, hover, pressed, selected, disabled, error, focus]
- Layout: [alineación, hug, fill, wrap, ancho disponible, etc.]
- Responsive: [cómo debe comportarse si no cabe]
- Variantes/properties: [qué debería poder editarse]
- Restricciones: [qué no debe hacer]
Completar solo las líneas que apliquen. No es obligatorio llenar las ocho.
Ejemplo aplicado
Referencia · selector de ordenamiento
Crear componente:
Selector de ordenamiento

Link:
[LINK DE FIGMA]

Especificaciones:
- Uso: ordenar el listado de compras.
- Contenido: label "Ordenar por" y opciones "Más reciente" / "Más antiguo".
- Comportamiento: desplegable seleccionable.
- Estados: default, abierto, hover, selected, disabled y focus.
- Layout: debe quedar alineado con los filtros superiores y dentro del espacio disponible.
- Responsive: no usar ancho fijo innecesario; debe adaptarse al contenedor con Hug contents / Fill container según corresponda.
- Restricciones: no debe salirse del frame ni romper el layout padre.
06

Ajustar un componente que quedó mal

Si algo sale mal, no se pide rehacer desde cero. Se pide ajuste sobre lo ya creado, apuntando al problema específico.

Prompt de ajuste
Chat "Componentes" · sobre el componente existente
Ajustar componente en Figma:

El componente quedó con [describir el problema, ej. ancho fijo y no se adapta al espacio disponible].

Link:
[LINK]

Corrige el componente directamente en Figma.

No lo recrees desde cero. Ajusta su comportamiento responsive.

Debe quedar:
- Dentro del contenedor disponible.
- Sin salirse del layout.
- Con auto layout correcto.
- Sin posiciones absolutas innecesarias.
- Con resize configurado según corresponda: Fill container / Hug contents.
- Si no cabe en la misma fila, debe bajar o ajustarse según el espacio.
- Mantener tokens, variables, estados, accesibilidad AA y la nomenclatura ya definida.

Revisa también el contenedor padre para que el componente se adapte correctamente al ancho disponible.
La primera línea es la única que cambia según el problema real observado.
07

Reforzar la regla de responsive

Cuando el componente vive dentro de una fila, header, filtro, card o tab, esta línea se agrega al final de cualquier prompt de creación o ajuste.

Línea de refuerzo
Agregar al final del prompt
Debe quedar dentro del espacio disponible del frame, sin ancho fijo innecesario y con auto layout responsive.

08

Checklist de revisión en Figma

Después de que Claude crea el componente, esto se revisa manualmente antes de documentarlo.

Layout

  • No se sale del frame.
  • No tiene ancho fijo innecesario.
  • Usa Auto Layout correctamente.
  • Funciona dentro del contenedor padre.
  • Se adapta al espacio disponible.

Estilos y consistencia

  • Usa los estilos ya definidos.
  • No usa colores hardcodeados.
  • Mantiene padding, gap, radius y tipografía.
  • Respeta la nomenclatura definida.
  • No duplica componentes existentes.

Interacción y estados

  • Tiene los estados necesarios.
  • Variantes/properties útiles, no excesivas.
  • El comportamiento es claro.
  • Mantiene accesibilidad AA.
  • Texto con legibilidad y contraste.
09

Buenas prácticas para ahorrar tokens

Reglas simples para que el equipo trabaje rápido sin gastar contexto repitiendo lo que el proyecto ya tiene configurado.

Hacer

  • Usar un chat exclusivo para componentes.
  • Entregar nombre del componente + link de Figma.
  • Agregar especificaciones solo cuando sea necesario.
  • Pedir ajustes sobre el mismo componente.
  • Reforzar responsive cuando el componente vive dentro de un layout complejo.

Evitar

  • Repetir todos los links de referencia en cada mensaje.
  • Pedir análisis cuando solo se necesita crear.
  • Dar instrucciones muy largas para componentes simples.
  • Pedir que lo rehaga desde cero si solo necesita ajuste.
  • Permitir anchos fijos sin justificación clara.
